# Review Candidates

Use this skill when an operator wants help triaging incoming ATS candidate cases.

Checklist:

1. Confirm the ATS workspace and target queue or job bucket.
2. Review the latest candidate cases, labels, notes, and current stage values.
3. Summarize the strongest and weakest candidates with explicit reasoning.
4. Propose safe stage changes, tags, and follow-up actions before applying them.
5. Keep auditability intact by writing notes instead of silently changing interpretation-heavy data.

Do not reject or close a candidate case without an explicit operator instruction.
